Table: orders
Schema: SALES
Connection: oracle_10

Description:
Stores order information with customer references and order status tracking.

Columns:
- order_id (NUMBER, Primary Key): Unique identifier for each order
- customer_id (NUMBER, Foreign Key → customers.customer_id): Reference to customer
- order_date (DATE, DEFAULT SYSDATE): Date when order was placed
- total_amount (NUMBER(10,2)): Total order amount in currency
- status (VARCHAR2(20), DEFAULT 'pending'): Order status (pending, processing, completed, cancelled, refunded)
- shipping_address (VARCHAR2(200)): Delivery address for this order
- payment_method (VARCHAR2(50)): Payment method used (credit_card, paypal, bank_transfer)
- tracking_number (VARCHAR2(100)): Shipment tracking number
- notes (VARCHAR2(500)): Additional order notes

Indexes:
- PK_ORDERS on order_id
- IDX_ORDER_CUSTOMER on customer_id
- IDX_ORDER_DATE on order_date
- IDX_ORDER_STATUS on status

Foreign Keys:
- FK_ORDER_CUSTOMER: customer_id → customers.customer_id

Related Tables:
- customers (customer_id references)
- order_items (order_id referenced by)
- shipments (order_id referenced by)

Example Queries:
-- Get all pending orders
SELECT * FROM orders WHERE status = 'pending' ORDER BY order_date DESC;

-- Get orders for a specific customer
SELECT * FROM orders WHERE customer_id = 12345 ORDER BY order_date DESC;

-- Get today's orders with customer names
SELECT o.*, c.first_name, c.last_name 
FROM orders o 
JOIN customers c ON o.customer_id = c.customer_id 
WHERE TRUNC(o.order_date) = TRUNC(SYSDATE);
