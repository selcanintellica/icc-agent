## Summary: Key Visual Concepts

### 1. Two Entry Paths
- **Generate Path**: User describes â†’ Agent generates SQL
- **Provide Path**: User writes SQL directly

### 2. Confirmation Before Execution
- Both paths require user confirmation before executing SQL
- Safety check to review SQL before running

### 3. Results Loop
- After showing results, user can:
  - Write to database (creates new job)
  - Email results (creates new job)
  - Done (exit)
- Write/Email actions loop back to results

### 4. Agent Specialization
- **SQL Agent** (7B code model, temp=0.1): Only for SQL generation
- **Job Agent** (8B general model, temp=0.3): Only for parameter extraction
- **Router**: Handles all flow logic (no LLM reasoning)

### 5. Connection Management
- UI provides connection names (user-friendly)
- Router converts to IDs (API-compatible)
- Transparent to user

### 6. Development vs Production
- Mock mode: Fast, no API, predefined tables
- Production mode: Real API calls for table definitions
